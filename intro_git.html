Introduction to Git and it's commands

29th March, 2022 | Sai Priya

Introduction

First of all, let's understand what is Version control system. Version control system is a tool which we developers use to record changes which we make either to a single file or to a set of files over a period of time so that we can obtain any specific versions of file easily.
It enables multiple people to simultaneously work on a single project. Each person can edit their own copy of files and finally share those changes with the rest of the team.
There are 3 types of version control systems available which are,

  • Local version control systems.
  • Centralized version control sytems.
  • Distributed version control systems.

What is GIT?

GIT is a distributed version control system, where each developer will have exact mirror image of the remote repository server. so even though connection to remote server is not available still the developer can do the required changes and she/he can commit those changes later, till then all the changes done by developer will be residing at local machine or local server which is located at each developer machine.

GIT Vs GitHub

Git is a version control tool which allows us to manage the project at local, also helps to fetch the data from central server/repository and allows to push the local machine data to the remote /central repository.


Whereas GitHub is an online service,or we can say an user friendly public hosting platform which allows to create and share the project in a remote server which helps the millions of developers to collaboratively work together to accomplish a single software product.
